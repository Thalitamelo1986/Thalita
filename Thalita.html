Repositorios GIT

Git

Pronunciado "guit"

A.Definicion:

Es un software de control de versiones disenado por Linus Torvalds, pensando en la eficiencia y confiabilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran numero de archivos de codigo fuente.

B.Lista de comandos basicos:

  1. git help: Muestra una lista con los comandos más utilizados en GIT.
  2. git init: Podemos ejecutar ese comando para crear localmente un repositorio con GIT y así utilizar todo el funcionamiento que GIT ofrece. Basta con estar ubicados dentro de la carpeta donde tenemos nuestro proyecto y ejecutar el comando. Cuando agreguemos archivos y un commit, se va a crear el branch master por defecto.
  3. git add + path: Agrega al repositorio los archivos que indiquemos.
  4. git add -A: Agrega al repositorio TODOS los archivos y carpetas que estén en nuestro proyecto, los cuales GIT no está siguiendo.
  5. git commit -m "mensaje" + archivos: Hace commit a los archivos que indiquemos, de esta manera quedan guardados nuestras modificaciones.
  6. git commit -am "mensaje": Hace commit de los archivos que han sido modificados y GIT los está siguiendo.
  7. git checkout -b NombreDeBranch: Crea un nuevo branch y automaticamente GIT se cambia al branch creado, clonando el branch desde donde ejecutamos el comando.
  8. git merge NombreDeBranch: Hace un merge entre dos branches, en este caso la dirección del merge sería entre el branch que indiquemos en el comando, y el branch donde estémos ubicados.
  9. git clone URL/name.git NombreProyecto: Clona un proyecto de git en la carpeta NombreProyecto.
  10. git push origin NombreDeBranch: Luego de que hicimos un git commit, si estamos trabajando remotamente, este comando va a subir los archivos al repositorio remoto, específicamente al branch que indiquemos.
  11. git pull origin NombreDeBranch: Hace una actualización en nuestro branch local, desde un branch remoto que indicamos en el comando.
